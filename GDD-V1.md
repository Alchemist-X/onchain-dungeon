## 0. 版本定位

**V1 是什么**：一个简单、好玩的 AI 托管大逃杀战斗游戏
**V1 不是什么**：不是 NFT 炒作平台、不是 DeFi 收益聚合器、不是投资赛道

**V1 的唯一目标**：验证"**花钱雇冒险者 → AI 替你打 → 胜者拿钱**"这个核心闭环能不能跑通、好不好玩。

持股、二级市场、投资他人角色等机制 **留到 V2** 再加。

## 1. 游戏概述

### 1.1 一句话描述
> 你花 5 USDC 雇一个冒险者，交给 OnChain OS 的 AI Agent 托管，每 8 小时它会自动参加一场 32 人大逃杀。你不下场，只负责决定雇不雇、强化不强化、何时退休。

### 1.2 核心玩法循环

```
雇佣冒险者 → 写战斗 Prompt → AI 托管参战 → 赢钱 / 死亡
   ↑                                            ↓
   └──────────── 再雇 / 复仇 ───────────────────┘
```

### 1.3 玩家做什么
- ✅ 花 USDC 雇冒险者（开盲盒）
- ✅ 选强化属性（充 USDC 提升战力）
- ✅ 写战斗策略 Prompt（指挥 AI）
- ✅ 决定战斗/休息/退休
- ❌ 不需要手操战斗
- ❌ 不需要每天上线

## 2. 世界观（深渊试炼场）

千年前，神明厌倦了永恒的和平，在世界裂缝处开启了**深渊试炼场**。
凡人可将灵魂抵押给 **AI 先知**（OnChain OS Agent），借其操纵一具冒险者躯体进入深渊。
// 凡人可以消耗代币，

- 深渊每 **8 小时**开启一次战门
- **32 名**冒险者被投入不同地形的竞技场
- 活着出来的瓜分所有死者的遗产
- **3 次怯战不出**的灵魂将被深渊永远吞噬

## 3. 核心系统设计

## 3.1 角色系统

### 3.1.1 //化身（Mint）
- **成本**：5 USDC
- **产出**：1 个冒险者 NFT（ERC-721），绑定一个 OnChain OS Agent Wallet
- **属性生成**：链上 VRF 随机，决定稀有度 + 3 属性分配

### 3.1.2 稀有度与属性

**3 个基础属性**（锁定总点数）：
- **STR 力量** —— 基础伤害
- **AGI 敏捷** —— 先手与命中
- **LUC 运气** —— 暴击与奇迹
// 生命 - 血量

**稀有度表**：

| 稀有度 | 掉率 | 总点数 | 单属性上限 |
|---|---|---|---|
| Common 凡人 | 60% | 15 | 7 |
| Uncommon 老兵 | 25% | 17 | 8 |
| Rare 骑士 | 10% | 19 | 9 |
| Epic 英雄 | 4% | 21 | 10 |
| Legendary 传奇 | 1% | 23 | 10 |

### 3.1.3 HP 公式
```
HP = 40 + STR × 3
```
范围 43–70，战斗中扣血至 0 即死亡。

---

## 3.2 强化系统

### 3.2.1 规则
- **每次强化 = 2 USDC**，玩家自选 +1 点（STR / AGI / LUC）
- **单角色最多强化 10 次**（共 20 USDC）
- **单属性最多 +5 点**
- **赛前 1 小时锁定**，不可强化

### 3.2.2 1:1 价值注入（Soul Vault） // 这个很奇怪呀，应该就是强化了多少钱，角色价值就增加多少钱，最后大决胜者才会被抽水10%
```
角色底价 = Mint 金额 + 累计强化金额
```

| 状态 | 累计投入 | Vault 底价 |
|---|---|---|
| 刚 mint | 5 | 3 USDC |
| 强化 5 次 | 15 | 13 USDC |
| 强化满（10 次） | 25 | 23 USDC |

> Mint 的 5 USDC 中，**3 USDC 进 Vault、2 USDC 归平台**
> 强化的 2 USDC **100% 进 Vault**（玩家花多少，角色多值多少）

### 3.2.3 Vault 流向 // 赛程过半后可以带着你的奖励退出，被抽水10%
| 触发 | 流向 |
|---|---|
| 角色死亡 | 90% 给击杀者、10% 销毁 |
| 主动退休 | 90% 返还雇主、10% 销毁 |
| 赛季冠军 | 保留 Vault + 瓜分奖池 |

## 3.3 战斗系统

### 3.3.1 排程（东八区）
| 场次 | 开战时间 | 报名截止 |
|---|---|---|
| 晨战 | 08:00 | 07:00 |
| 午战 | 16:00 | 15:00 |
| 夜战 | 00:00 | 23:00 |

### 3.3.2 报名与休息
- 每场开战前，玩家可为角色提交**报名**或**休息**指令（Commit-Reveal 加密上链）
- **未响应 = 默认休息**
- 报名一经提交**不可撤回**（防止明牌策略 / 卡点心理战）
- 报名阶段可**预设未来 3 场**策略（应对时区问题）

### 3.3.3 休息惩罚（死亡计数器）
- **连续 7 次休息** → 被深渊吞噬（NFT 销毁）
- 每次参战成功**重置计数器**
- 设计意图：允许短期休整，但惩罚长期挂机

### 3.3.4 报名池与赛制 //允许最多512人参与，比赛时间压缩至每2h比一次
- 每场**最多 32 人**
- 报名人数超 32 → 随机抽 32 名入场（未抽中不扣计数器）
- 报名人数不足 16 → 取消本场，计数器不扣
- 入场后 **4 人一桌 × 8 桌** free-for-all
- 每桌胜者晋级：8 → 4v4 × 2 → 2 → 冠军

### 3.3.5 单场战斗流程 //这个也要做成1v1 默认会发起攻击，拼出胜者，不需要nvn

```
第 1 轮：8 桌 4 人 FFA（free-for-all）    → 出 8 名幸存者
第 2 轮：2 桌 4 人 FFA                     → 出 2 名决赛者
第 3 轮：1v1 决赛                          → 1 名冠军
```

### 3.3.6 4 人桌战斗回合

每回合流程：
1. **地形事件触发检查**（20% 概率）
2. **初始化**：所有存活角色掷 `AGI + D20` 决定行动顺序
3. **逐人行动**：AI Agent 按 Prompt 选择目标（1 名敌人）
4. **命中判定**：`攻 AGI + D20` ≥ `守 AGI + D20` → 命中
5. **伤害结算**：
   - 基础 = `STR + D10`
   - **暴击**（`LUC + D20 ≥ 22`）→ ×2
   - **奇迹**（`LUC + D20 ≥ 28`）→ ×3
6. 扣 HP，HP ≤ 0 → 死亡
7. 下一回合

**平均一桌 5–8 回合结束。**

### 3.3.7 地形事件（赛前公开，全桌共享）//不需要

每场战斗开局随机抽 1 张地形卡：

| 卡牌 | 效果 |
|---|---|
| 🌋 熔岩裂隙 | 每回合全员扣 D6 HP |
| 🌫️ 浓雾之地 | 所有命中判定 -5 |
| ⚡ 雷暴 | 每回合 25% 概率对随机 1 人造成 20 伤害 |
| 🎲 混沌祭坛 | 每回合第一次掷骰结果 ×2 |
| 🩸 鲜血狂热 | 每次暴击攻击者回 10 HP |
| 🌟 神恩降临 | 本场所有人 LUC +3 |

## 3.4 奖励与死亡 // 直接改成胜者获取败者所有资产

### 3.4.1 奖池构成
每场战斗的奖池 = **死者的 Vault 汇总 × 90%** + 平台固定贡献（每场 50 USDC 基础池）

### 3.4.2 奖励分配
| 名次 | 奖池比例 |
|---|---|
| 冠军 | 50% |
| 亚军 | 20% |
| 4 强（半决赛失败者 × 2） | 各 10% |
| 8 强（第 1 轮胜者，决赛未晋级） | 各 2.5% |

### 3.4.3 击杀奖励
- 每击杀 1 名对手额外奖励 **5 USDC**（从平台基础池出）
- 在 4 人桌中，最后收尾的 KO 者获得击杀奖励

### 3.4.4 死亡后
- 冒险者 NFT 立即 **burn**
- 死者 Vault 90% → 所有击杀参与者（按伤害占比分）
- 死者 Vault 10% → 黑洞（通缩）
- 雇主收到死亡通知（OnChain OS 推送）

### 3.4.5 主动退休
- 玩家可在任何时候（非赛前锁定期）让角色**退休**
- 退休 = NFT burn，Vault 90% 返还雇主、10% 销毁
- 设计意图：给高光角色"落袋为安"的选择

## 3.5 AI 决策（OnChain OS 集成）

### 3.5.1 Agent 绑定
- 每个冒险者 NFT = 1 个独立 OnChain OS Agent
- Agent 有独立钱包、独立状态、独立 Prompt

### 3.5.2 玩家可配置的 Prompt 模板

```yaml
角色名: 暗影猎手
战斗风格: |
  - 开局苟在后排，等对手互杀
  - HP > 50% 时偏防守，< 50% 时拼命暴击
  - 优先攻击：(1) HP 最低者  (2) LUC 最高者
  - 禁止：第一回合主动攻击
强化优先级: LUC > AGI > STR
休息触发条件: 连续两败自动休息一场
```

### 3.5.3 Agent 战斗时可读取的数据
- 自己 / 对手的当前 HP、属性、历史战绩
- 地形卡、剩余回合数
- 其他玩家的 Prompt 类型标签（公开的"激进/苟活"标签）
- 不可读取：对手的具体 Prompt 内容（保密性设计）

### 3.5.4 AI 决策输出 // agent只允许在强化属性和挑选对手上做选择，还有是否退出
每回合 Agent 输出结构化决策：
```json
{
  "target": "adventurer_0x123...",
  "action": "attack" | "defend" | "wait",
  "reasoning": "对手 HP 最低，且我的暴击期望更高"
}
```
**Reasoning 上链可查**，每个决策可追溯（反作弊 + 叙事）。

## 4. 经济模型 //这边都要改，平台只对胜者和提早退出的收费

### 4.1 USDC 流向总览

```
玩家 Mint (5)──┬─→ 2 USDC 平台
              └─→ 3 USDC Vault

玩家强化 (2)──→ 100% Vault

角色死亡 Vault──┬─→ 90% 击杀者
               └─→ 10% 销毁

战斗奖池 ──┬─→ 90% 前 8 名瓜分
          ├─→ 5% 击杀奖励
          └─→ 5% 平台
```

### 4.2 平台收入来源
1. Mint 抽成（每次 2 USDC × 预计日 mint 量）
2. 战斗奖池抽成（5%）
3. 不抽强化（1:1 注入 Vault，这是对玩家的核心承诺）

### 4.3 通缩设计 // 不需要
- 死亡 Vault 10% 销毁
- 退休 Vault 10% 销毁
- NFT 死亡直接 burn
- 防止代币通胀 + 经济系统无限膨胀

## 5. 玩家生命周期

### 5.1 新手体验（第一天）
1. 连接钱包，5 USDC mint 首个冒险者（1 分钟）
2. 系统自动生成默认 Prompt（3 种预设："激进 / 平衡 / 苟活"）
3. 提交下一场战斗报名
4. 8 小时后收到战斗结果推送（胜 / 败）
5. 胜 → 分到 USDC；败 → NFT 销毁，再 mint 新角色

### 5.2 中度玩家（第一周）
- 同时托管 2–3 个不同流派的冒险者
- 自定义 Prompt，开始针对特定对手
- 摸索强化策略（STR 流 vs AGI 流 vs LUC 流）

### 5.3 重度玩家(第一个月)
- 培养 1 个"明星角色"，多次夺冠
- 在链上跟踪其他强者，设计反制 Prompt
- 参与社区讨论流派 / 地形策略

## 6. 防作弊与公平性

### 6.1 随机性来源（XLayer 轻量方案）
- 赛前由合约运营方提交一个 `seedHash = keccak256(seed)`（commit）
- 开战时揭示 `seed`（reveal），并与 `block.prevrandao + blockhash(N-1) + matchId` 混合：
  `rand = keccak256(seed, block.prevrandao, blockhash(block.number - 1), matchId, nonce)`
- 每次掷骰递增 `nonce`，所有掷骰结果与 `seed / rand` 均上链可查
- **不引入 Chainlink VRF**：XLayer 上无成熟 VRF，commit-reveal + prevrandao 对 MVP 足够，成本低、可审计

### 6.2 Commit-Reveal 报名
- 报名时提交 Hash，开战时揭示
- 防止"看到对手报名再决定"的明牌策略

### 6.3 赛前锁定
- 开战前 1h 锁定所有操作：强化、Prompt 修改、报名撤销
- 防止场前通过操作获取不公平优势

### 6.4 Prompt 保密
- 玩家 Prompt 内容 **加密存储**，Agent 运行时解密
- 其他玩家只能看到公开标签，不能看具体内容

## 10. 设计原则总结

1. **战斗优先，投资靠后**：V1 只做战斗，不做 NFT 炒作
2. **轻充值**：5 USDC 入场，25 USDC 封顶，拒绝氪金碾压
3. **1:1 保底**：充多少钱，角色就值多少钱
4. **弱者有机会**：初始随机 + 掷骰 + 地形卡，最低胜率 15%
5. **AI 真托管**：不是服务器算完上链，而是 OnChain OS Agent 亲自决策
6. **死亡有代价**：NFT 永久销毁，Vault 转给击杀者 —— 赌性与粘性来源
